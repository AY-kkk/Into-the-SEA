# 商业化设计（Gate3）

## 1. 免费 / 付费边界

| 能力                        | 免费版     | 专业版（¥29/月） |
| --------------------------- | ---------- | ---------------- |
| 招录情报 / 申论案例浏览     | ✅ 全量    | ✅ 全量          |
| 行测刷题                    | 每日 50 题 | 不限量           |
| 模拟面试 / 岗位检索 AI 调用 | 每日 10 次 | 每日 200 次      |
| 练习进度云同步              | ✅         | ✅               |
| 报告 / 错题本导出           | —          | ✅               |
| 新功能优先体验              | —          | ✅               |

套餐定义唯一真源：`src/lib/billing/plans.ts`（前端定价页、API 配额护栏、成本模型共用）。

## 2. AI 用量分层

- 每日 LLM 调用配额按套餐区分，在 `src/lib/security/guard.ts` 中通过 `getPlan(user.plan).limits.dailyLlmCalls` 传入 `consumeLlmQuota`。
- 单次请求 `max_tokens` 上限由 `LLM_MAX_TOKENS_CAP` 全局护栏约束，防止单请求爆量。
- 超额返回 429 并附 `upgrade` 提示，引导免费用户升级。

## 3. 单用户成本模型

实现见 `src/lib/billing/cost-model.ts`（可用 `COST_AVG_TOKENS_PER_CALL` / `COST_PRICE_PER_1K_TOKENS` 覆盖假设）。

默认假设：单次调用约 1400 tokens（受 `LLM_MAX_TOKENS_CAP=1024` 封顶），单价约 ¥0.004/千 tokens。

| 套餐 | 每日上限 | 满额月成本(最坏) | 典型月成本(25%) | 月费 | 典型毛利         |
| ---- | -------- | ---------------- | --------------- | ---- | ---------------- |
| 免费 | 10 次    | ≈¥1.23           | ≈¥0.31          | ¥0   | 获客成本，可接受 |
| 专业 | 200 次   | ≈¥24.6           | ≈¥6.1           | ¥29  | 正毛利           |

> 结论：专业版在典型使用率下毛利为正；免费版成本极低，作为获客与转化入口可持续。实际接入真实计费后，用 env 覆盖假设即可重算。
